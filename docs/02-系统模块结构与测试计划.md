# 系统模块结构与测试计划
# System Module Structure & Testing Plan

## 1. 系统模块概览 System Module Overview

```
lab-management-system/
├── backend/                    # 后端服务
│   ├── app/
│   │   ├── api/v1/endpoints/   # API端点 (18个模块)
│   │   ├── models/             # 数据模型 (15个模型)
│   │   ├── schemas/            # Pydantic验证 (16个模式)
│   │   ├── services/           # 业务逻辑层
│   │   └── core/               # 核心配置
│   ├── alembic/                # 数据库迁移
│   └── tests/                  # 单元测试 (14个测试模块)
│
├── frontend/                   # 前端应用
│   └── src/
│       ├── pages/              # 页面组件 (23个页面)
│       ├── components/         # 可复用组件 (18个模态框+)
│       ├── services/           # API客户端
│       ├── stores/             # 状态管理
│       ├── hooks/              # 自定义Hooks
│       └── utils/              # 工具函数
│
└── docs/                       # 项目文档
```

## 2. 后端模块详情 Backend Module Details

### 2.1 API端点模块 API Endpoint Modules

| 模块 | 文件 | 端点前缀 | 功能描述 |
|------|------|----------|----------|
| 认证 | `auth.py` | `/api/v1/auth` | 登录、注册、令牌管理 |
| 用户 | `users.py` | `/api/v1/users` | 用户CRUD、角色管理 |
| 站点 | `sites.py` | `/api/v1/sites` | 站点信息管理 |
| 实验室 | `laboratories.py` | `/api/v1/laboratories` | 实验室管理 |
| 人员 | `personnel.py` | `/api/v1/personnel` | 人员信息、技能、借调 |
| 设备 | `equipment.py` | `/api/v1/equipment` | 设备管理、调度 |
| 技能 | `skills.py` | `/api/v1/skills` | 技能定义、分配 |
| 工单 | `work_orders.py` | `/api/v1/work-orders` | 工单全生命周期管理 |
| 物料 | `materials.py` | `/api/v1/materials` | 物料追踪、库存 |
| 客户 | `clients.py` | `/api/v1/clients` | 客户信息、SLA |
| 仪表板 | `dashboard.py` | `/api/v1/dashboard` | 统计数据、KPI |
| 交接 | `handovers.py` | `/api/v1/handovers` | 班次交接 |
| 方法 | `methods.py` | `/api/v1/methods` | 分析/测试方法 |
| 班次 | `shifts.py` | `/api/v1/shifts` | 班次定义 |
| 审计 | `audit_logs.py` | `/api/v1/audit-logs` | 操作审计 |
| 权限 | `permissions.py` | `/api/v1/permissions` | RBAC权限管理 |
| 报表 | `reports.py` | `/api/v1/reports` | 数据导出 |

### 2.2 数据模型 Data Models

| 模型 | 文件 | 主要字段 | 关联关系 |
|------|------|----------|----------|
| User | `user.py` | id, username, email, role, password | → Laboratory, Site |
| Site | `site.py` | id, name, code, address, contact | ← Laboratory, Equipment |
| Laboratory | `laboratory.py` | id, name, code, lab_type, site_id | → Site, ← Equipment, Personnel |
| Personnel | `personnel.py` | id, employee_id, user_id, status | → User, Laboratory, Site |
| Equipment | `equipment.py` | id, name, code, type, status | → Laboratory, Site |
| Skill | `skill.py` | id, name, code, category | ← PersonnelSkill |
| WorkOrder | `work_order.py` | id, order_number, title, status | → Laboratory, Client, Personnel |
| WorkOrderTask | `work_order.py` | id, task_number, status | → WorkOrder, Equipment |
| Material | `material.py` | id, material_code, name, status | → Laboratory, Client, WorkOrder |
| Client | - | id, name, code, contact | ← WorkOrder, Material |
| AuditLog | `audit_log.py` | id, action, entity, user_id | → User |
| Permission | `permission.py` | id, role, permission | - |
| Handover | `handover.py` | id, from_user, to_user, status | → User |
| Method | `method.py` | id, name, code, category | - |
| Shift | `shift.py` | id, name, start_time, end_time | → Laboratory |
